  1. 작업 개요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ■ 목적
    NCS 사업관리 분야의 능력단위(51개)를 O*NET 직업(1,016개)과 의미적으로
    비교하여, 각 능력단위에 가장 가까운 O*NET 직업 후보군을 자동으로 생성.

  ■ 핵심 방법론
    Sentence-BERT 임베딩 + 코사인 유사도 행렬 (51 × 1,016)

  ■ 실행 노트북
    notebooks/02_similarity_mapping.ipynb

  ■ 입력 데이터
    - data/raw/NCS_DB.xlsx              (NCS 사업관리 원본)
    - data/processed/onet_processed.csv (O*NET 전처리 결과)

  ■ 출력 데이터
    - data/processed/ncs_units_translated.csv       (능력단위 번역 텍스트)
    - data/processed/embeddings/ncs_unit_embeddings.npy  (51 × 384)
    - data/processed/embeddings/onet_embeddings.npy      (1016 × 384)
    - data/processed/embeddings/sim_matrix_unit_onet.npy (51 × 1016)
    - outputs/mapping_candidates_unit.csv           (능력단위별 Top-20)
    - outputs/mapping_candidates_detail.csv         (세분류별 Top-20)
    - outputs/mapping_candidates_sub.csv            (소분류별 Top-20)
    - outputs/ncs_onet_mapping_candidates.xlsx      (멀티시트 통합본)

  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 2-1. NCS 텍스트 재구성 전략                                             │
  └─────────────────────────────────────────────────────────────────────────┘

  기존 전처리(Step 1)의 소분류(2개) 수준 텍스트(수십만 자)는 임베딩 모델의
  입력 토큰 한도(256 tokens)를 크게 초과하므로, 능력단위(51개) 수준으로
  세분화하여 재구성.

  [텍스트 구성 원칙]
  ① 원본 데이터의 반복 행에서 중복 제거(dedup) 후 필드별 고유 값만 수집
  ② O*NET의 데이터 구조(Tasks / Knowledge / Skills)와 대응되도록 섹션 구성:

    "{능력단위명칭} ({세분류}, {소분류}).
     Tasks: {능력단위요소명 (unique)}.
     Performance criteria: {수행준거 (unique)}.
     Knowledge: {지식 KSA (unique)}.
     Skills: {기술 KSA (unique)}.
     Attitudes: {태도 KSA (unique)}."

  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 2-2. 번역                                                               │
  └─────────────────────────────────────────────────────────────────────────┘

  도구 : deep_translator.GoogleTranslator (ko → en)
  전략 : 능력단위당 500자 단위 청크 분할 후 번역, 실패 시 원문 유지
  캐시 : data/processed/ncs_units_translated.csv 저장으로 재실행 방지


  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 2-3. 임베딩 모델                                                        │
  └─────────────────────────────────────────────────────────────────────────┘

  모델     : sentence-transformers/all-MiniLM-L6-v2
  출처     : HuggingFace Hub (Apache 2.0 라이선스)
  파라미터 : 22M
  출력 차원: 384
  최대 토큰: 256 tokens (초과 시 자동 truncation)
  정규화   : normalize_embeddings=True (L2 정규화)
             → 코사인 유사도 = 내적(dot product) 으로 단순화

  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 2-4. 유사도 계산 및 집계                                                │
  └─────────────────────────────────────────────────────────────────────────┘

  유사도 계산 : sklearn.metrics.pairwise.cosine_similarity
               → 행렬 연산 (51 × 1,016 = 51,816 쌍)

  집계 방법 (세분류 / 소분류 수준):
    세분류 유사도 = 소속 능력단위들의 유사도 행 평균 (mean pooling)
    소분류 유사도 = 소속 능력단위들의 유사도 행 평균 (mean pooling)

  추출 기준 : Top-20 후보 (능력단위 / 세분류 / 소분류 각 수준별)

  ┌─────────────────────────────────────────────────────────────────────────┐
  │ 9-3. 개선 3 — NCS 청크 임베딩 (Chunk Embedding)                        │
  └─────────────────────────────────────────────────────────────────────────┘

  문제 : NCS 능력단위 텍스트 평균 4,144자(≈1,036 tokens)에서 모델 한도인
          256 tokens 이후 내용이 전부 truncation → 전체 텍스트의 ~75% 무시.
  해결 : 텍스트를 900자(≈225 tokens, 안전 마진 31 tokens) 단위 청크로 분할,
          각 청크를 독립 임베딩 후 평균 풀링하여 L2 재정규화.

  알고리즘:
    1) text를 900자씩 슬라이딩 (overlap 100자)
    2) 각 청크 → 384-dim 벡터 (normalize_embeddings=False)
    3) 청크 벡터 산술 평균 → mean_vec
    4) mean_vec / ‖mean_vec‖₂ → 단위 벡터 (최종 임베딩)

  효과 : 능력단위당 평균 약 4개 청크 처리 → 전체 텍스트 정보 활용 가능

  │ 9-4. 개선 4 — TF-IDF 앙상블 (Keyword Similarity Ensemble)              │
  └─────────────────────────────────────────────────────────────────────────┘

  문제 : Sentence-BERT는 의미적 유사도를 잘 포착하나 전문 키워드(도메인 용어)
          에는 상대적으로 민감도가 낮음.
  해결 : TF-IDF 기반 키워드 유사도를 보조 신호로 결합.

  설정:
    TfidfVectorizer(max_features=15000, ngram_range=(1,2), sublinear_tf=True)
    최종 유사도 = 0.65 × 임베딩 코사인 유사도 + 0.35 × TF-IDF 코사인 유사도

  유니그램 + 바이그램으로 "project management", "financial risk",
  "human resources" 등 복합 직무 용어를 별도 피처로 확보.
