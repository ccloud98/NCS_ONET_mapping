  NCS와 O*NET은 직무 역량을 기술하는 방식과 분류 체계가 다르다.
  의미 있는 비교를 위해 각 체계의 구성 요소를 기능적으로 대응시켜
  4개의 공통 비교 차원을 정의한다.

  ┌──────────────────────────────────────────────────────────────────────────┐
  │ 차원         │ NCS 구성 요소          │ O*NET 구성 요소             │ 의미 │
  ├──────────────────────────────────────────────────────────────────────────┤
  │ [A] 업무활동 │ 능력단위요소명         │ Work Activities (IM>=3.5)   │ 직무 수행에 반복 사용되는 고수준 행동 범주 │
  │ [T] 수행과업 │ 수행준거               │ Task Statements             │ 구체적 직무 과업 — 가장 직접적 비교 차원 │
  │ [K] 지식     │ 지식 KSA               │ Knowledge (IM>=3.5)         │ 필요 이론·원리·분야 지식 │
  │ [S] 기술     │ 기술 KSA               │ Skills (IM>=3.5)            │ 방법론·도구·역량 — 체계 차이 가장 큰 차원 │
  └──────────────────────────────────────────────────────────────────────────┘

  [차원별 대응 근거]

  업무활동 [A]
    NCS의 능력단위요소는 "~하기" 형태로 기술된 상위 행동 단위.
    O*NET Work Activities는 "Getting Information", "Making Decisions and Solving Problems"
    등 직업 전반에 걸쳐 반복되는 고수준 행동 범주.
    양측 모두 '직무를 위해 어떤 활동을 하는가'를 기술하는 차원.

  수행과업 [T]
    NCS 수행준거는 "~할 수 있다" 형태의 검증 가능한 구체적 수행 기준.
    O*NET Task Statements는 "~한다(direct, manage, analyze)" 형태의 직무 기술.
    양측 모두 실제 수행하는 과업을 가장 직접적으로 기술하는 차원.

  지식 [K]
    NCS 지식 KSA와 O*NET Knowledge 모두 직무에 필요한 이론·원리·정보 체계를
    기술. 두 체계가 동일 개념("지식/Knowledge")을 사용하는 유일한 차원.
    학문 영역·분야 키워드로 비교.

  기술 [S]
    NCS 기술 KSA는 도구·소프트웨어·절차 수행 능력 중심.
    O*NET Skills는 "Critical Thinking", "Active Listening", "Systems Analysis"
    등 인지·사회·기술적 역량 중심.
    분류 체계 자체가 달라 가장 큰 구조적 차이가 발생하는 차원.

  [유사도 측정 방법]
    - 방법: TF-IDF (bigram, sublinear_tf=True, stop_words=english) 코사인 유사도
    - 각 차원별 독립 측정 -> 4개 점수 프로파일 생성
    - 공통 키워드: 양측 TF-IDF 상위 10개 교집합
    - 전용 키워드: 한쪽에만 존재하는 고유 개념
    - 데이터: NCS 번역 텍스트 (clean_text_en) 섹션 파싱 + O*NET 원본 영어 텍스트
    - 임베딩 종합 점수: Sentence-BERT 청크 임베딩 + TF-IDF 앙상블 (65%:35%)
